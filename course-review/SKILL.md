# 课程评审 Skill

## 核心理念

审阅不是"找错"，是**在交付前做一次系统性检查**——识别结构性缺口、跨阶段断裂、内容深度不足和可教性风险。

## 输入

用户提供**课程项目目录的路径**。评审程序扫描目录下所有产出物文件，不要求完整的 Phase 00→05 流程——有部分文件也能审。

## 工作流程

本 Skill 按以下 5 个维度依次执行。每个维度独立评估，不依赖前一个维度的结论。

### 维度 1：结构完整性

检查产出物流是否完整，每个阶段预期产出物是否存在。

**操作**：扫描课程项目目录，对照以下标准检查：

| 产出一览 | 说明 |
|:---|:---|
| `00-*.md` | Phase 00（探索阶段）产出，如存在则评估其内容 |
| `01-*.md` | Phase 01 产出：需求来源、痛点验证、学习者画像、能力差距 |
| `02-*.md` | Phase 02 产出：学习目标、情绪曲线、对齐矩阵 |
| `03-*/` | Phase 03 产出目录：大纲/学员手册/讲师手册/实操案例/测评/课件 |
| `04-*.md` | Phase 04 产出：前置测评、随堂验证设计 |
| `05-*.md` | Phase 05 产出：柯氏评估、迭代计划 |

**每一份产出物都应包含"本阶段完成检查"清单**——检查项应为已勾选状态（[x]），未勾选的需在评审中标明。

> 详细检查项见 `references/review-methodology.md` → 结构完整性

### 维度 2：内部一致性

验证跨阶段的信息是否一致——这是传统质检清单最常遗漏的维度。

**操作**：挑三个关键字段，沿产出物链追踪比对：

| 检查项 | 追踪路径 | 典型断裂 |
|:---|:---|:---|
| **目标人群** | Phase 01 画像 → Phase 02 目标难度 → Phase 03 案例场景 → Phase 04 测评 | 画像写"零基础毕业生"，案例却假设 2 年经验 |
| **核心痛点** | Phase 00/01 痛点 → Phase 02 惊叫点设计 → Phase 03 讲师话术 → Phase 05 复盘 | 讲师手册没有针对焦虑设计话术 |
| **学习目标** | Phase 02 目标清单 → Phase 03 测评方案 → Phase 05 达成度 | OBJ 在 Phase 02 定义了，Phase 03 没有对应测评 |

**方法**：从 Phase 01 开始，沿产出物流往下看，每到一个文件确认"这个文件里的信息是否还认同上一个文件里的同一个字段"。

> 详细检查项见 `references/review-methodology.md` → 内部一致性

### 维度 3：内容深度

比 "文件存在" 更进一步——内容是否达到了 **可教学** 的质量。

| 深度层级 | 定义 | 判断方法 |
|:---|:---|:---|
| **L1 骨架级** | 有文件、有结构、有占位符 | 文件能打开，结构完整 |
| **L2 填充级** | 内容完整，可读性强 | 逐段阅读，没有"待填写"标记 |
| **L3 教学级** | 拿到就能上课，讲师不需额外准备 | 模拟一节课——只看手册，能讲吗？ |
| **L4 精品级** | 有视觉设计、有录屏、有课后扩展 | 学员体验完整，超出预期 |

**课程交付时应至少达到 L2**，关键模块（如 Day 1 的 M1）应达到 L3。

> 详细检查项见 `references/review-methodology.md` → 内容深度

### 维度 4：可教性

模拟"明天就要上课"场景，验证材料是否可执行。

| 检查项 | 操作方式 |
|:---|:---|
| **时间表可执行** | 把课程大纲的每段时间加起来，检查是否等于总时长 |
| **讲师能照着讲** | 打开讲师手册，从 M1 到结束逐段读——"这里有盲区吗？" |
| **案例可操作** | 还原案例场景——决策点有明确选项吗？参考答案有层次差异吗？ |
| **测评可评分** | 拿 3 份不同水平作业试评——评分标准能区分优/中/差吗？ |
| **技术依赖确认** | 实操环节用到的工具/平台/API 是否都已确认可运行？ |

### 维度 5：风险评估

识别课程交付前最可能出问题的 3 件事：

```
风险 1：[如"M4 自由手搓依赖 AI 工具可用性"]
  影响：工具不可用 → 整个 M4 叙事弧断裂
  概率：低
  应对：备本地演示录像降级

风险 2：[描述]
  影响：...
  概率：...
  应对：...

风险 3：[描述]
  影响：...
  概率：...
  应对：...
```

**风险来源**：讲师经验（没讲过的环节）、技术依赖（AI 工具、API）、学员差异（跨度大的分层）、内容时效性（过时的案例）。

## 产出

完成评审后，按以下格式输出评审报告：

```markdown
## 课程质量评审报告：[课程名称]

### 总体评估
完整性: /5 | 一致性: /5 | 深度: L1-L4 | 可教性: /5 | 风险: 高/中/低

### 各维度发现
- 维度 1（结构完整性）：[发现]
- 维度 2（内部一致性）：[发现]
- 维度 3（内容深度）：[发现]
- 维度 4（可教性）：[发现]
- 维度 5（风险评估）：[发现]

### 待办清单（按优先级）
| 优先级 | 问题 | 建议 | 涉及文件 |
|:---|:---|:---|:---|
| P0 | | | |
| P1 | | | |
| P2 | | | |
```
