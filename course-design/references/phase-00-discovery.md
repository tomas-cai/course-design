# Phase 00：培训需求发现

本文档用于在 Phase 01 分析阶段之前，从模糊的个人观察和市场信号中结构化地发现培训商机。

## 适用场景

- 你有一些分散的行业观察，想判断"哪个方向值得做一门课""
- 你接触了很多目标用户，觉得"他们需要被培训"，但不确定具体是什么方向
- 你想在投入课程开发之前，系统性地评估一个培训方向的商业价值

## 第一阶段：信号采集（发散）

用头脑风暴的方式，不加判断地列出所有你观察到的信号。分三个维度：

### 市场信号

引导问题：你观察到市场上发生了什么变化？

| 维度 | 引导问题 |
|:---|:---|
| **技术趋势** | 有什么新技术正在改变工作方式？ |
| **政策/法规** | 有没有新的合规要求？ |
| **竞争者动态** | 同行/竞品在推什么课程？什么方向越来越多人讨论？ |
| **招聘市场** | 什么岗位需求激增？什么技能被频繁提及？ |

### 用户信号

引导问题：你从目标人群身上看到了什么？

| 维度 | 引导问题 |
|:---|:---|
| **用户痛点** | 他们在什么场景下说"好难""好烦""不知道怎么办"？ |
| **自学行为** | 他们在自学什么？（搜什么关键词、问什么问题、看什么内容） |
| **效率差距** | 做得好的和做得不好的人之间，差的是什么？ |
| **职场焦虑** | 他们在焦虑什么？（怕被替代、怕跟不上、怕选错方向） |

### 组织信号

引导问题：你所看到的组织/团队正在被什么困扰？

| 维度 | 引导问题 |
|:---|:---|
| **能力断层** | 团队需要什么能力但招不到/培养不出来？ |
| **效率瓶颈** | 什么环节最耗时、最容易出错、最依赖个人经验？ |
| **工具变革** | 团队引入了什么新工具但用不起来？ |
| **目标变化** | 公司的战略方向变了，团队需要什么新能力？ |

## 第二阶段：信号聚类 → 痛点提取（收敛）

### 信号分组

把采集到的信号按"可能属于同一个培训方向"进行分组。每个组至少 3 条信号才能作为一个候选方向。

| 候选方向 | 相关信号 | 初步判断 |
|:---|:---|:---|
| 方向 A（如"AI 原型设计"） | 信号 1, 3, 5 | 可能是一个培训机会 |
| 方向 B | 信号 2, 4 | 只有 2 条信号，需补充 |

### 候选方向多维对比

如果候选方向 >= 2 个，用以下表格做结构化对比，再决定优先推进哪个：

| 对比维度 | 方向 A | 方向 B | 方向 C |
|:---|:---|:---|:---|
| 痛点等级 | L1-L3 | L1-L3 | L1-L3 |
| 培训可解？ | ✅ / ❌ | ✅ / ❌ | ✅ / ❌ |
| 市场容量（1-5） | | | |
| 竞品强度（1-5=低） | | | |
| 我方优势（1-5） | | | |
| 直觉判断 | ⭐/⭐⭐/⭐⭐⭐ | ⭐/⭐⭐/⭐⭐⭐ | ⭐/⭐⭐/⭐⭐⭐ |
| **综合结论** | 推进 / 暂缓 / 放弃 | 推进 / 暂缓 / 放弃 | 推进 / 暂缓 / 放弃 |

|:---|:---|:---|
| 方向 A（如"AI 原型设计"） | 信号 1, 信号 3, 信号 5 | 可能是一个培训机会 |
| 方向 B | 信号 2, 信号 4 | 只有 2 条信号，需补充 |

### 痛点分级

对每个候选方向，评估其痛点的强度：

- **L3 紧迫痛点**：用户已经在找解决方案了，不解决就有明确的业务损失
- **L2 成长痛点**：用户知道应该提升但还没有紧迫感，解决了有显著价值
- **L1 兴趣痛点**：用户觉得"学一下也不错"但优先级不高

### 区分"培训能解决的" vs "培训不能解决的"

| 可培训 | 不可培训 |
|:---|:---|
| 知识/技能缺失 | 组织/流程/文化问题 |
| 方法论/框架不清晰 | 工具采购决策 |
| 判断力/决策能力不足 | 团队激励/薪酬问题 |
| 认知盲区/思维模式 | 商业模式/战略决策 |

## 第三阶段：商机评估

对每个候选方向（至少 L2 以上），做商机评估：

| 评估维度 | 问题 | 评分（1-5） |
|:---|:---|:---|
| **市场容量** | 潜在学员有多少？公司愿意为此付费吗？ | / 5 |
| **付费意愿** | 学员或公司愿意付多少钱？ | / 5 |
| **竞争强度** | 市场上已有多少同类课程？质量如何？ | / 5（低竞争=高分） |
| **内容稳定性** | 这个方向的内容多久会过时？ | / 5（越稳定越高分） |
| **我方优势** | 我们在这个方向有什么独特的积累或资源？ | / 5 |
| **总评分** | 加权平均 | / 5 |

> 如果总评分 < 3.0，建议暂缓这个方向。

## 第四阶段：课程假设形成

### 课程假设 → Phase 01 输入映射

课程假设中的每个字段都直接对应 Phase 01 的输入。完成本阶段后，
`00-培训需求spec.md` 可直接作为 Phase 01 的"用户提供的需求文档"：

| Phase 00 字段 | 对应 Phase 01 输入 | 说明 |
|:---|:---|:---|
| 暂定课程名 | 课程定位决策 | Phase 01 可在此基础上升级 |
| 一句话价值主张 | 业务需求表述 | 转化为"课程要解决的差距" |
| 目标人群（初步） | 学习者画像 | Phase 01 需细化为详细画像 |
| 核心痛点（初步） | 痛点验证 | Phase 01 需做量化和分层 |
| 课程类型假设 | 课程定位决策 | Phase 01 需确认 |
| 交付形式假设 | 课程定位决策 | Phase 01 需确认 |
| 为什么是我们做 | 需求来源 | 说明课程方的独特价值 |
| 不做的风险 | — | 辅助 Phase 01 判断优先级 |



对通过商机评估的方向，形成"一页纸课程假设"：

```
## 课程假设：暂定名

### 1. 一句话价值主张
[这门课帮助什么人解决什么问题？]

### 2. 目标人群（初步）
[谁最需要这门课？]

### 3. 核心痛点（初步）
[他们最痛的 1-3 个点是什么？]

### 4. 课程类型假设
认知课 / 技能课 / 思维课

### 5. 交付形式假设
[线上/线下/混合，预估时长]

### 6. 为什么是我们做？
[我们在这个方向有什么独特的优势？]

### 7. 不做的风险
[如果现在不做，会错过什么？]
```
