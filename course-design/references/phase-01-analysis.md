# 分析阶段参考（Phase 01 - Analysis）

## 节奏选择

进入本阶段时，先判断用户状态来决定执行节奏：

| 用户特征 | 使用模式 | 做法 |
|:---|:---|:---|
| 消息中包含多个信息点，能一次性回答多个问题 | **批量模式**（默认） | 将下面的核心问题一次性抛出，用户集中回答后统一处理 |
| 消息简短、表示不确定、需要引导 | **逐题模式** | 按下面的探索问题逐个深入追问 |

### 批量模式：核心问题

- 课程名称与定位
- 目标人群（有哪些类型？各有什么特征？）
- 核心痛点（学员不解决这个问题的损失是什么？迫切程度？）
- 核心能力目标（上完课能做什么？）
- 课程时长与交付形式
- 课程类型（认知课/技能课/思维课）

### 逐题模式：深入探索

- 需求来源细节与触发场景
- 非培训问题标记
- 每类学员的学习动机与情绪基线
- 先决技能的具体要求
- 能力差距的数字细化（当前→目标 1-5 分）

## 第一节：业务需求拆解

思路：像产品经理写 PRD 一样拆解培训需求——需求不拆透，设计必跑偏。

### 问题清单（向用户逐一提问）

1. **需求来源**
   - 提出方是谁？（业务方 / 学员代表 / 公司战略 / 其他）
   - 原始需求描述是什么？（引导用户一字不差复述原话）
   - 什么场景或事件触发了这个需求？

2. **痛点验证**
   - 目标用户是否已经明确知道自己有问题？
   - 这个问题如果不解决，会有什么业务损失？（量化：时间/成本/质量）
   - 用户是否尝试过自行解决？如果试过，用的什么方法？效果如何？
   - 目标用户对这个问题有多迫切？（1-5 分）

3. **业务目标 → 学习目标转换**
   - 引导用户逐一列出"业务需求"→"能力缺口"→"课程要解决的差距"
   - 标记出"不适合通过培训解决的"问题（如：工具采购决策、团队激励、流程问题）——课程不能背着不该背的锅

4. **课程定位决策**
   - 课程类型：认知课 / 技能课 / 思维课？
   - 交付形式：线上录播 / 线上直播 / 线下工作坊 / 混合式？
   - 期望时长与完课率目标？
   - 是否认证课程？
   - 课程优先级：P0 / P1 / P2？

### 常见误区
- ❌ 把"领导想开这门课"当作需求分析的全部
- ❌ 跳过痛点验证直接进入设计
- ❌ 把组织和流程问题包装成培训需求

## 第二节：学习者画像与情绪分析

### 结构化的学习者画像维度

1. **基本信息**：岗位、经验年限（新手/中级/高级）、团队规模
2. **知识起点**：对课程主题的先决知识水平（1-5 分）
3. **工作场景**：在实际工作中会用到课程内容的具体场景
4. **学习动机**：
   - 主动学习（想提升自己）还是被动学习（被安排来上课）？
   - 期待什么收获？
5. **情绪起点**：
   - 对课程主题的态度是什么？兴奋 / 焦虑 / 无所谓 / 抵触？
   - 如果有焦虑，来源是什么？（怕学不会、怕内容没用、怕耽误时间）
6. **学习习惯与偏好**：
   - 偏好理论讲解还是动手实操？
   - 偏向自学还是讲师引导？
   - 可投入的学习时间？

### 情绪地图（关键方法）

画出学习者在课程前的"情绪基线"来判断设计策略：

| 情绪状态 | 设计策略 |
|:---|:---|
| **焦虑高**（怕学不会） | 降低认知门槛，增加信心建设环节，设计"小胜利" |
| **期待高**（想学但不知道学什么） | 快速确认价值，给出课程全景图，避免失望 |
| **无所谓**（被安排来的） | 先做痛点唤醒，制造认知冲突，建立学习必要感 |
| **抵触**（觉得没用/浪费时间） | 用真实案例证明相关性，让学员自己发现"我缺这个" |

## 第三节：能力差距诊断

### 分层诊断框架

用布卢姆分类法逐层评估当前 vs 目标状态：
> **多学员类型处理**：如果课程面向多类学员（如新人 PM + 有经验 PM），
请为每类学员分别评估，将表格扩展为多列（"新人当前→目标 | 有经验当前→目标"）。
每类的差距和优先级可能不同，应分别标记。


| 能力维度 | 当前水平（1-5） | 目标水平（1-5） | 差距 | 优先级 |
|:---|:---|:---|:---|:---|
| 知识层（记忆/理解） | | | | P0-P3 |
| 应用层（应用/分析） | | | | P0-P3 |
| 高阶层（评价/创造） | | | | P0-P3 |

### 先决技能检查

判断学员是否具备学习该课程所需的基础：
- 需要什么先决知识或技能？
- 目标人群是否普遍具备？
- 如果不具备，是否需要设计前置预习材料？

### 产出

完成分析阶段后输出：**课程需求与定位报告**（1-2 页），包含：
1. 需求来源与痛点验证结论
2. 学习者画像（含情绪基线）
3. 能力差距诊断结果
4. 课程定位决策
5. 非培训问题的标记与建议
6. P1 验证计划（如有需要）
