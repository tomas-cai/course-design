# 磨课 (Course Design)

> 把课程当作产品做——学习者就是用户，教学就是体验设计，而不仅是知识传递。

**磨课** 是一套课程开发全生命周期 Skill，基于 **ADDIE × 迪克-凯瑞 × 布卢姆 + LXD（学习体验设计）+ 敏捷迭代 + 柯氏四级评估** 的六维嵌套模型。

每个阶段产出标准化文件，下一阶段以此为输入，支持中途接续和跨项目复用。

---

## 核心理念

### 六维嵌套模型

| 层次 | 维度 | 解决的问题 |
|:---|:---|:---|
| **流程层** | ADDIE | 项目管控 — 无章法、返工率高 |
| **质量层** | 迪克-凯瑞 | 内容对齐 — 教的不考、考的不用 |
| **精度层** | 布卢姆目标分类法 | 目标模糊 — "了解/掌握"全凭感觉 |
| **体验层** | LXD + 产品思维 | 学习动机与完课率 — 痛点/惊叫点/爽点 |
| **敏捷层** | SAM/AGILE | 内容过时 — AI 领域周期长则失效 |
| **评估层** | 柯氏四级 | 效果闭环 — 学了≠会用，会用≠业务改善 |

### 痛点 / 惊叫点 / 爽点

- **痛点**：学习者在学习前/学习中遇到的真实困难、焦虑、挫败
- **惊叫点**：让学习者发出"原来如此！"的顿悟时刻，打破旧认知
- **爽点**：让学习过程顺畅、有掌控感的细节体验

### 课程即产品的阶段映射

| 课程开发阶段 | 产品开发类比 |
|:---|:---|
| 分析阶段 | 用户研究 + 需求定义 |
| 设计阶段 | 产品方案 + 交互设计 |
| 开发阶段 | 研发 + 测试 |
| 实施阶段 | 上线 + 运营 |
| 评估阶段 | 数据分析 + 迭代 |

## 产出物流（Artifact Chain）

磨课的核心设计：每个阶段产出唯一的标准化文件，下一阶段以此为输入。

```
<course-project-root>/
│
├── 01-需求定位报告.md             ← Phase 1 产出 → Phase 2 读取
├── 02-课程设计方案.md             ← Phase 2 产出 → Phase 3 读取
├── 03-教学材料包/                 ← Phase 3 产出 → Phase 4 读取
│   ├── 00-课程大纲.md
│   ├── 01-学员手册.md
│   ├── 02-讲师手册.md
│   ├── 03-实操案例.md
│   └── 04-测评方案.md
├── 04-实施记录与反馈.md           ← Phase 4 产出 → Phase 5 读取
└── 05-评估报告与迭代计划.md       ← Phase 5 产出（完结）
```

每个产出物是自包含的 Markdown 文件，含元数据头和内容正文。下一阶段读取它就能知道上一阶段做了什么决策，不需要外部上下文。

## 工作流程

### 入口与阶段选择

| 场景 | 行为 |
|:---|:---|
| 从头开发 | 创建项目目录 → Phase 01 分析阶段 |
| 已有产出物 | 检出文件 → 从对应阶段继续 |
| 从中间切入 | 用户提供已有文档 → 读取后从指定阶段开始 |
| 跨项目参考 | 读另一个课程的评估报告作为输入 |

### 五个阶段

| Phase | 名称 | 核心产出 | 核心问题 |
|:---|:---|:---|:---|
| 01 | 分析阶段 | `01-需求定位报告.md` | 学习者的痛点是什么？为什么愿意花时间学？ |
| 02 | 设计阶段 | `02-课程设计方案.md` | 惊叫点在哪里？学习旅程中有没有认知高潮？ |
| 03 | 开发阶段 | `03-教学材料包/` | 四侧材料（学员/讲师/实操/测评）是否到位？ |
| 04 | 实施阶段 | `04-实施记录与反馈.md` | 学员的爽点是什么？节奏是否舒适？ |
| 05 | 评估阶段 | `05-评估报告与迭代计划.md` | 痛点假设复盘 + 迭代方向 |

### 执行节奏

每个阶段支持两种执行模式，自动判断：

- **批量模式**（默认）：用户能一次性回答多个问题，核心问题一次抛出、统一处理
- **逐题模式**：用户表示不确定，逐个深入追问

## 适用场景

- **通用课程开发**：技能课、认知课、思维课均可
- **PM + AI 方向**：判断类能力目标编写、案例驱动测评、内容时效性分层管理
- **标准化技能培训**：无需过度使用体验设计层，但方法论框架同样适用

### PM + AI 课程的特殊处理

| 维度 | 适配方式 |
|:---|:---|
| 判断类能力 | 布卢姆高层级（分析→创造）为主要标尺；案例判断 + 同行评议替代传统考试 |
| 内容时效性 | 底层原理（年更）/ 工具方法论（季更）/ 实操示例（月更）三层管理 |
| 测评设计 | 模拟场景题、案例拆解题、方案评估、项目答辩 + 评分标准 |
