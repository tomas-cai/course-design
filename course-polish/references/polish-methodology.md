# 课程润色方法论详细参考

## 维度 1：叙事弧润色

### 每课时的"起承转合"模板

| 环节 | 时长占比 | 讲师做什么 | 学员感受 | 润色要点 |
|:---|:---:|:---|:---|:---|
| **起** | 10% | 扔一个问题/一个反常案例 | "为什么？" | 不要用"大家好今天我们来学…"，用"先看一个有意思的现象…" |
| **承** | 40% | 建立认知框架，给方法论 | "原来是这样" | 拆成 3 个以内的小节，每节讲完后加一句话总结 |
| **转** | 20% | ⭐ 创造一个反差/揭晓答案 | "原来如此！" | 先让学员猜，再给答案。猜错的人记忆最深 |
| **合** | 30% | 联系实际 + 布置作业 | "我可以试试" | 给一个具体的"下一步动作"，不要说"大家可以想想" |

### 惊叫点触发技巧

| 类型 | 话术模板 |
|:---|:---|
| **反转** | "你们以为答案是 A？——其实答案是 C。原因是…（停顿）…" |
| **对比** | "这是自由搓的——现在看规范驱动的——同一个需求。（停顿）" |
| **揭密** | "质量最好的那位学员用的方法不是 Prompt 写得好——（停顿2秒）——他写了一本操作手册。" |
| **自我揭秘** | "我自己第一次做这个 demo 的时候也翻车了——你们猜哪一步最坑？" |

## 维度 2：话术润色

### 平铺 vs 润色对照表

| 原话（平铺） | 润色后 | 用了什么技巧 |
|:---|:---|:---|
| "AGENTS.md 是把你的项目规则告诉 AI" | "你不是在跟 AI 聊天——你是在给它发一本操作手册。这本手册就是 AGENTS.md。" | 画面感 + 对比 |
| "自由手搓的目的是体验没有规范时的状态" | "先让你自己摔一跤，摔了才知道痛在哪。然后我们再教你走路。" | 画面感 + 隐性承诺 |
| "这门课学完后你会掌握 AI 原型设计的能力" | "2 周后，有人问你『AI 产品怎么做』——你打开 GitHub 就可以直接演示。" | 具体场景 |
| "作业是写一份 AI PRD，写在 GitHub 上" | "作业：想象你下周要跟老板汇报这个 AI 功能——你总不能说『我还没想好』吧？把 PRD 写好，这就是你的汇报底稿。" | 紧迫感 + 场景 |
| "大家有没有问题？" | "在 5 秒内想一个问题——我能答就答，答不出的我记下来课后回复。" | 降低提问门槛 |
| "这节课我们讲 AI PRD 的写法" | "面试官问『你的 AI PRD 和传统 PRD 有什么不同』——你怎么回答？" | 从问题开始 |
| "好，我们开始下一节" | "休息之前——记住一句话就够了：…（一句话概括）" | 强收束 |

### 润色检查清单

- [ ] 每段讲师话术是否以"问题"或"反常现象"开头，而不是以"今天我们来学…"开头？
- [ ] 抽象概念是否有一句画面感的类比？
- [ ] 每段讲授后是否有 1 句话的总结（学员可以抄下来的那种）？
- [ ] 关键信息是否重复了 3 遍（用不同的说法）？
- [ ] 有没有"我觉得…""一般来说…"这类减弱语气的话？（去掉更自信）

## 维度 3：节奏设计

### 不同时段的节奏策略

| 时段 | 学员状态 | 推荐活动 | 避免 |
|:---|:---|:---|:---|
| 09:00-09:30 | 刚坐下,未进入状态 | 提问开场 + 快速破冰 | 放 PPT 标题页 + 念课程简介 |
| 09:30-11:00 | 专注力高峰 | 核心讲授 + 互动 | 安排练习（打断了专注流） |
| 11:00-12:00 | 血糖下降,注意力下降 | 动手练习 + 小组讨论 | 继续讲授 |
| 13:30-14:30 | 午饭后昏沉 | ⚡ 实操 + 站着的活动 | PPT 讲授 + 坐着不动 |
| 14:30-16:00 | 回升 | 轻量讲授 + 讨论 | 学新概念（认知负荷高） |
| 16:00-18:00 | 疲劳高峰 | 收尾 + 答疑 + 预告明天的亮点 | 讲新内容 |

### 节奏应急

| 症状 | 药方 |
|:---|:---|
| 学员频繁看手机/打哈欠 | **立刻切换活动**——不要试图用"讲大声点"解决。切换成举手投票/同桌讨论/快速写作 |
| 学员盯着屏幕不动 | **放一段对比案例**——人的注意力在"找不同"时最集中 |
| 连续有人问同一个问题 | **说明这是共性问题**——"刚才有 3 位同学问了同一个问题，说明我没讲清楚——重新说一遍" |

## 维度 4：互动设计

### 互动类型速查

| 你要达成什么 | 用什么互动 | 时长 | 话术模板 |
|:---|:---|:---:|:---|
| 快速摸底 | 举手投票 | 1min | "觉得 A 方案更好的举手——好——觉得 B 更好的举手——" |
| 引出观点 | 同桌讨论 | 3min | "跟旁边的人讨论 2 分钟：你觉得这个方案最大的问题在哪——2 分钟，开始" |
| 检验理解 | 冷 call | 1min | "我待会会问一个人——不是为难你，是检验我讲清楚了没有" |
| 巩固记忆 | 快速写作 | 2min | "用一句话概括刚才讲的三个区别——30 秒——写下来" |
| 制造惊叫点 | 错误展示 | 5min | "这是一位同学提交的作业——你们找找问题在哪" |
| 回顾前情 | 同伴教学 | 3min | "用 1 分钟把上节课的内容讲给你的同桌听" |

### 冷场应对

| 场景 | 破解话术 |
|:---|:---|
| "大家有没有问题？"——没人回答 | 换成"我数 5 秒，想一个问题——能答的我现场答，答不出的我记下来" |
| 讨论没人说话 | 换成"先自己写 30 秒，再跟同桌说"——写比说容易 |
| 提问学员答不上来 | "没关系——我换个方式问——" 不要让他尴尬地站着 |

## 维度 5：临场应变

### 时间失控

| 情况 | 行动 |
|:---|:---|
| **快了** | 展开标注为"可展开"的讨论题；增加 Q&A |
| **慢了** | 砍掉标注为"可跳过"的段落；压缩互动时间 |
| **严重超时** | 缩短优秀作业分享；压缩冲刺辅导 |

### 技术故障（AI 工具不可用等）

1. 不要慌——学员比讲师更宽容
2. 启动降级方案（录播演示 / 讲师代操作 / 抛话题讨论）
3. 一边操作一边说："这是最好的教学时刻——正好让你看到工具出问题时 PM 应该怎么应对"

### 学员质疑

| 质疑 | 应对 |
|:---|:---|
| "这个我工作中用不上" | "好问题——能不能告诉我你具体在什么场景下工作？我们一起想想有没有适配的角度" |
| "这个太简单了" | "好——那你试试能不能做得更快——我计时" |
| "这个太难了" | "你说得对——第一遍是有点绕——我换个方式讲第二遍" |

### 课后润色复盘

润色不是一次性的。每次讲完后记录：
- 哪个环节学员最投入？→ 保留
- 哪个环节学员走神最多？→ 调整
- 哪个问题学员问得最多？→ 说明没讲清楚，下次改话术

## 润色版标记规范

输出润色版讲师手册时，使用以下标记：

| 标记 | 含义 | 示例 |
|:---|:---|:---|
| 🔥 | 叙事弧建议 | 新开场方式/高潮点位置调整 |
| ✨ | 话术润色 | 原话→润色版的逐句替换 |
| ⏱ | 节奏调整 | 该段建议加速/减速/切换活动 |
| 🎯 | 互动设计 | 建议插入的互动环节 + 话术 |
| 🚨 | 应急标注 | 此段可跳过/可展开 |
