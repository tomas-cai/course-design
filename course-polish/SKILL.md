# 课程润色 Skill

## 核心理念

**同一句话，换种说法，冲击力完全不同。** 课程润色不做内容层面的调整（不改教学目标、不删知识点），只做**体验层面的优化**。

如同电影剪辑——剧本不改，但剪辑节奏、配乐、镜头语言决定了观众是否入戏。

## 工作流程

本 Skill 按以下 5 个维度依次对讲师手册进行润色。每个维度独立操作，不依赖前一个维度的结论。

### 维度 1：叙事弧润色（Story Arc）

每个课时应该有一条微型的"起承转合"线。检查讲师手册中每个课时的结构：

| 阶段 | 时长占比 | 作用 | 缺失表现 |
|:---|:---:|:---|:---|
| **起**（钩子） | ~10% | 扔一个问题/一个案例，让学员产生"为什么？" | 直接进入主题，没有引发好奇 |
| **承**（铺陈） | ~40% | 建立认知框架，给出方法论 | 信息点平铺，没有主次 |
| **转**（高潮） | ~20% | ⭐ 制造一个"原来如此"时刻 | 全程没有起伏，学完就忘 |
| **合**（收束） | ~30% | 联系实际 + 布置作业 | 截然而止，没有升华 |

> 详见 `references/polish-methodology.md` → 叙事弧润色

### 维度 2：话术润色（Script Polish）

把平铺直叙的讲师话术改写成更有画面感、更容易记住的表达。

| 原话 | 润色后 |
|:---|:---|
| "AGENTS.md 是把你的项目规则告诉 AI" | "想象一下，你给 AI 画了一条跑道。你不是在跟它聊天，是在给它发一本操作手册。这本手册就叫 AGENTS.md。" |
| "自由手搓的目的是让你体验没有规范时的状态" | "先让你自己摔一跤——摔了才知道痛在哪。然后我们再教你走路。" |

**润色原则**：
1. **画面感**——不要说概念，说画面（"发一本操作手册"比"告诉规则"有画面）
2. **对比**——好的表达建立在对比之上（"摔跤 vs 走路"）
3. **悬念**——先扔问题，再给答案（"你知道吗？"）
4. **短句**——一句话不要超过 25 个字

> 详见 `references/polish-methodology.md` → 话术润色

### 维度 3：节奏设计（Energy Rhythm）

同一个讲师手册，早上讲和下午讲、课前讲和课后讲，效果完全不同。节奏润色根据**该课时的当天时间 + 学员精力状态**调整：

| 时段 | 学员状态 | 设计策略 |
|:---|:---|:---|
| 09:00-09:30 | 清醒但未进入状态 | **用问题开场**，不用"大家好今天我们来学…" |
| 09:30-11:00 | 专注力高峰 | **核心内容**安排在此 |
| 11:00-12:00 | 开始疲劳（血糖下降） | **互动环节**替代讲授 |
| 13:30-14:30 | 午饭后昏沉 | **动手实操**替代听讲（坐着不动更困） |
| 14:30-16:00 | 回升 | **轻量内容 + 讨论** |
| 16:00-18:00 | 疲劳高峰 | **收尾 + 明天预告**，不要安排新内容 |

> 详见 `references/polish-methodology.md` → 节奏设计

### 维度 4：互动设计（Engagement Design）

不是"有没有互动"的问题，是"学员想不想参与"的问题。

| 互动类型 | 适用场景 | 设计要点 |
|:---|:---|:---|
| **冷call** | 检验理解 | 不要叫"那个穿蓝衣服的同学"——提前说"我待会会问一个问题，随机叫一位同学" |
| **同桌讨论** | 引出观点 | 给明确问题 + 30s 计时，不要"大家讨论一下"（太宽泛） |
| **举手投票** | 快速摸底 | "觉得 A 方案更好的举手——觉得 B 更好的举手"——比"大家觉得怎么样"有效 |
| **快速写作** | 巩固记忆 | "用一句话总结刚才我讲的三个区别，30 秒——开始"——不要只让学生"想" |
| **错误展示** | 制造惊叫点 | 展示一个真实的错误范例，让其他学员发现"问题在哪"——比自己讲解效果好 10 倍 |

> 详见 `references/polish-methodology.md` → 互动设计

### 维度 5：临场应变（Live Contingency）

给讲师手册补充"如果…怎么办"的应急标注：

| 场景 | 应对 |
|:---|:---|
| **时间不够** | 标出可以砍掉的部分（"此段跳过不影响主线"） |
| **时间过剩** | 标出可以展开的讨论题（"如果时间充裕，可以加这个案例"） |
| **冷场** | 自问自答脚本："我在准备时也想过这个问题…你们有没有类似的？" |
| **学员偏离话题** | "好问题，把它记下来我们课后聊——现在先回到主线" |
| **技术故障** | "我们先用录播演示代替" |

> 详见 `references/polish-methodology.md` → 临场应变

## 产出

完成润色后，在讲师手册同一目录下输出 `02-讲师手册·润色版.md`。

润色版不替代原版——它是在原版基础上用标记增补：

```
🔥 叙事弧改进建议
✨ 话术润色版本
⏱ 节奏调整建议
🎯 互动环节设计
🚨 应急标注
```

原版讲师手册保持不变，润色版作为增补参考。
